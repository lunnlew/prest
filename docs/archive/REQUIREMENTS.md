## 2. 编辑器核心功能详细需求

### 2.1 Monaco Editor 集成 (E-001)

**需求描述：** 集成 Monaco Editor 作为 Markdown 编辑器核心

**验收标准：**
- [ ] 编辑器正确加载并显示
- [ ] 支持文本输入和编辑
- [ ] 编辑器实例可被其他组件访问
- [ ] 编辑器配置（字体、行高）可通过设置调整

**实现文件：** `src/components/editor/MonacoEditor.tsx`

**测试方法：**
1. 启动应用，确认编辑器加载
2. 输入文本，确认编辑功能正常
3. 通过设置面板调整字体大小，确认生效

---

### 2.2 Markdown 语法高亮 (E-002)

**需求描述：** 为 Markdown 文本提供语法高亮显示

**验收标准：**
- [ ] 标题（# 标记）显示特定颜色
- [ ] 粗体（**text**）高亮显示
- [ ] 斜体（*text*）高亮显示
- [ ] 代码块（```）高亮显示
- [ ] 链接（[]()）高亮显示
- [ ] 列表（-、*、1.）高亮显示

**实现方式：** 自定义 Monarch tokens provider

**实现文件：** `src/components/editor/MonacoEditor.tsx`

**测试方法：**
1. 输入各种 Markdown 元素
2. 检查各元素是否有正确的颜色区分

---

### 2.3 实时编辑 (E-003)

**需求描述：** 编辑内容实时同步到状态管理

**验收标准：**
- [ ] 输入立即更新 store
- [ ] 预览区实时反映编辑内容
- [ ] 无延迟或卡顿

**实现方式：** 监听 editor.onDidChangeModelContent

**实现文件：** `src/stores/editorSlice.ts`

**测试方法：**
1. 输入文本
2. 检查预览区是否同步更新

---

### 2.4 撤销/重做 (E-004)

**需求描述：** 支持编辑操作的撤销和重做

**验收标准：**
- [ ] Ctrl+Z 撤销上一步操作
- [ ] Ctrl+Y/Ctrl+Shift+Z 重做操作
- [ ] 撤销栈最多保存 50 步
- [ ] 状态栏显示可撤销/重做数量

**实现方式：**
- 使用 Monaco Editor 内置 undo/redo
- Zustand 维护独立 undoStack/redoStack

**实现文件：** `src/stores/editorSlice.ts`

**测试方法：**
1. 输入多段文本
2. 按 Ctrl+Z 多次撤销
3. 按 Ctrl+Y 重做
